Linux下的一些配置文件说明

以下为在CentOS8上的配置， 有些也在Arch上做了测试

/etc/profile

启动login-bash时会读取的配置文件, 并且只有login-bash才会读取。

/etc/bashrc

两种登入方式都会读取的配置，所有用户都会读取的bash配置 并且 /etc/profile 会调用这个文件 所以默认情况下这个文件会被读取两次 但是Arch发行版默认没有 /etc/bashrc 取代的是 /etc/bash.bashrc 文件并且由 /etc/profile 来调用， 而且不会自己调用

用户的bash配置文件

.bash_profile

启动login-bash时会读取的配置文件，并且只有login-bash才会读取。 是用户级别的配置文件.

.bash_login

很少会用到, 只有不存在 ~/.bash_profile 文件时才会使用该文件

.profile

很少会用到, 只有不存在 ~/.bash_profile ~/.bash_login 文件时才会使用该文件

.bashrc

login-bash和non-login-bash都会读取的bash配置 login-bash是通过 .bash_profile 来调用的 non-login-bash默认自己会读取 .bash 文件

xserverrc

X Server启动加载的配置文件 优先级: .xserverrc /etc/X11/xinit/xserverrc 单纯的运行/usr/bin/X

xinitrc

X Client启动加载的配置文件 优先级: .xinitrc /etc/X11/xinit/xinitrc 单纯的运行xterm 由xinit程序执行

Xauthority

用于X使用的认证

Xmodmap

xmodmap 的键位配置文件

xprofile

用于随会话自动运行程序，或从显示管理器启动,尤其是那个会话没有自带自动启动程序功能时 - 比如一个独立的窗口管理器。 使用gdm等图形登录时读取并运用里面的设置

Xsession

登录管理器会使用.Xsession来初始化登录

Xsessionrc

debian系的一个特性，不是debian的系同没有它。

Xresource

一些X软件的设置, 显示设置 如 xterm xurvt rofi

Xdefaults

和 Xresource 一样但现在很少使用了

xdg目录

一些X上的软件全局配置放到这个目录
