记录Arch启动检测磁盘失败解决

原因

在不正常关机时会在再次启动系统的时候进行磁盘自检, 但是该自检有的时候会无法通过.

Failed to start File System Check on /dev/sda4
See 'Systemctl status systemd-fask@dev-sda4.service'
...
(or press Control-D to continue):

出现上述情况可通过以下方法解决:

  1. 对于这种系统告诉了你出现错误的是哪一个硬盘的情况使用fsck -y /dev/sda4然后reboot就可以解决
  2. 对于不知道哪里出了问题的情况使用fsck然后在随后的对话框中输入:y重启就可以解决
-y: 对所有问题默认回答yes
-t: 只检测指定文件系统类型的分区 例如: fsck -t ext4 /dev/sda4 等效于 fsck.ext4 /dev/sda4

使用fsck是要注意该磁盘分区是没有挂载的不然会有问题

更详细的fsck使用可以man fsck fsck --help 或 看这里
