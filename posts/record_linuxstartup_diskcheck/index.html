记录Arch启动检测磁盘失败解决

2019.10.23

原因

不清楚是什么原因, 但网上说不正常关机会出现自检不正确的情况 

Failed to start File System Check on /dev/sda4
See 'Systemctl status systemd-fask@dev-sda4.service'
...
(or press Control-D to continue):

解决

  1. 对于这种系统告诉了你出现错误的是哪一个硬盘的情况使用 fsck -y /dev/sda4 然后 reboot 就可以解决

  2. 对于不知道哪里出了问题的情况使用 fsck 然后在随后的对话框中输入 :y 重启就可以解决

使用fsck是要注意该 磁盘分区是没有挂载 的不然会有问题 

-y: 对所有问题默认回答yes
-t: 只检测指定文件系统类型的分区 例如: fsck -t ext4 /dev/sda4 等效于 fsck.ext4 /dev/sda4

更详细的fsck使用可以 man fsck fsck --help 或 看这里
